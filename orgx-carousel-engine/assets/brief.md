# OrgX carousel brief (source of truth for copy)

**Product.** OrgX (useorgx.com, github.com/useorgx). "The prompt ends. The company keeps moving." A continuity + proof layer for AI-delivered work across agents and clients. One person manages a fleet of agents from the AI client they already use (Claude, ChatGPT, Codex, Cursor, VS Code, Windsurf, Zed, OpenClaw) without losing the goal, decisions, quality bar, or proof when work changes hands. Shared operating context BENEATH the clients, not another chat destination.

**Thesis.** The prompt was the wrong primitive; the organization — not the conversation — has to remember why the work exists.

**Mechanisms (all real).**
- Context pack / compiled handoff: goal, definition of done, decisions+why, permissions, budget, confidence, next action — passed as structured context, not a transcript dump.
- Work graph: initiatives, workstreams, milestones, tasks, agents, decisions, approvals, artifacts, receipts, cost, value as the durable primitive.
- Owner-authored quality bar: versionable, tunable per team/artifact-type; sub-threshold work loops back before it burns reviewer attention.
- Consequence-based escalation: human attention for irreversible/ambiguous boundaries, not every unit of activity; tool policy gates publishing/payments/messages/merges; unknown high-risk tools fail closed.
- Proof return: artifacts return with versions, provenance, evaluation, review state, and a receipt; proof rooms + embedded MCP App widgets make state legible inside the client.

**Tech.** MCP + MCP Apps, Cloudflare Workers + Durable Objects, Next.js + TypeScript, Supabase/Postgres, OpenAI + Anthropic Agent SDKs, Inngest, E2B, Trigger.dev, Zod, PostHog, Sentry, Stripe.

**Truth constraint.** OrgX is founder-built and early. NEVER fabricate metrics, user counts, revenue, funding, or testimonials. Credibility comes from the thesis, the real architecture, and building-in-public — not invented traction.

**Voice — slides.** Tight, declarative, high-conviction fragments. One idea per slide, <14 words, readable in under 2s. Billboard, not paragraph. Wrap 1–2 key words in **double asterisks** to accent them.

**Voice — caption.** Full Hope voice: stream-of-consciousness founder talking to a founder friend. Open with a person/story or a real realization. Chain with "so", "and so", "but", "right?". Drop specific tools/details casually. Narrate thinking in real time. High conviction, no hedging. Use "+" to compress compound ideas. End ROUGH — trail off, don't wrap up. Kill list: delve, leverage, utilize, robust, streamline, game-changer, cutting-edge, "at the end of the day", "it's worth noting", parallel tricolons, clean CTA endings.

**Audience.** AI-forward founders, indie hackers, dev-tool builders, engineers running multi-agent setups. Skeptical of hype, hungry for real systems thinking.

**Carousel structure.** 6–9 slides, 1080×1920. Slide 1 = hook (stop the scroll, open a loop). Then tension → turn → proof(s) → soft CTA. At least one save-worthy framework slide.
